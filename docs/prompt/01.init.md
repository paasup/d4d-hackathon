10시간이라는 극도의 제한된 시간 내에 **에지 COP(수십 개 모킹) - 중앙 플랫폼(대시보드 + 프라이빗 AI)** 간의 데이터 및 명령 순환 구조를 완벽히 시연하기 위한 최소 기능 제품(MVP) 기준의 기술 요건 정의(Functional & Technical Requirements)입니다.

이 요건은 실제 무거운 인프라 설치 시간을 최소화하고, 파스업(PAASUP) 위에서 구현 가능한 수준으로 경량화(Lightweight)하면서도 심사위원에게 시각적 임팩트를 주도록 설계되었습니다.

## 1. 에지 COP (Edge COP Simulator) 기술 요건

실제 하드웨어 대신 단일 머신(또는 가상환경)에서 **수십 개의 분산 에지 단말의 거동을 동시에 모킹하는 경량 프로그램**으로 구현합니다.

* **기능 요건:**
* **다중 에지 에이전트 모킹:** 최소 20~30개의 가상 에지 COP 가 고유 ID, 위경도 좌표, 탄약 수(0~100%)를 가지고 독립적으로 구동.
* **주기적 상태 송신:** 5~10초 주기로 중앙 COP 에 JSON 포맷으로 상태 전송 (무선망 특성을 감안해 페이로드 최소화).
* **하달 명령 수신 및 상태 천이:** 중앙에서 '보급 명령'을 수신하면 즉시 탄약 상태를 '보급 중'으로 변경하고, 일정 시간(예: 10초) 후 '보급 완료(탄약 100%)' 상태를 다시 중앙으로 역전송.


* **추천 기술 스택:**
* **Language:** Python (Asyncio 또는 Threading 라이브러리로 수십 개 에이전트 동시 구동)
* **통신 프로토콜:** Websocket (실시간 양방향) 또는 FastAPI 기반의 Lightweight REST API

## 2. 중앙 COP (Central Intelligence Platform) 기술 요건

### 2.1) 에지 상태 수집 및 다차원 시각화 (COP Dashboard)

수십 개의 에지 데이터를 실시간으로 지도 위에 뿌려주고, 줌인/줌아웃에 따른 **계층적(Hierarchical) 상황판**을 시각화합니다.

* **기능 요건:**
* **실시간 데이터 적재 및 브로커:** 에지들이 쏘는 실시간 데이터를 메모리(또는 초경량 DB)에 유실 없이 적재.
* **클러스터링 및 줌인/줌아웃:** * **광역 뷰(Zoom-out):** 사단/여단급 관할 구역 단위로 에지 COP들의 상태를 요약(예: 빨간색/초록색 핀 표시).
* **상세 뷰(Zoom-in):** 특정 에지 COP 영역을 확대하면, 그 하위의 관리 대상(예: GOP-21 분초, K9 자주포 개별 위치)과 각각의 잔여 탄약 수가 분리되어 표현됨.


* **사이드바 상세 정보:** 특정 에지 핀을 클릭하면 화면 우측에 실시간 잔여 탄량 그래프와 보급 진행 상태(대기/이동중/보급완료) 플래그 표시.


* **추천 기술 스택:**
* **Frontend/Dashboard:** Streamlit (Python 기반으로 3시간 내 대시보드 구축 가능) 또는 Python Folium(지도 시각화) + Plotly
* **Backend:** FastAPI



### 2.2) 교본/매뉴얼 기반 자율 진단 및 액션 추천 (Rule/LLM Trigger)

에지의 상태 변화를 감지하여 시스템이 자율적으로 위기를 경고하고 교본 기반 대응책을 팝업으로 띄웁니다.

* **기능 요건:**
* **임계치 기반 트리거(Rule Engine):** 특정 에지의 탄약 수 수치가 20% 이하로 떨어지는 순간, 시스템 이벤트가 발생하며 대시보드에 **'⚠️ [GOP-21] 탄약 고갈 위기'** 알림 발생.
* **컨텍스트 매뉴얼 매핑:** 위기 감지 시 미리 준비된 텍스트 교본 파일(`국방_군수지원_교본.txt`)에서 해당 상황(탄약 부족)에 맞는 프로토콜을 시스템이 즉각 매칭.
* **액션 추천 UI:** 지휘관 화면에 **"교본 제3조에 의거, B보급기지로부터 탄약 운반차 2대 배정을 추천합니다. [하달하기] "** 버튼을 포함한 추천 컴포넌트 렌더링.


* **추천 기술 스택:**
* Python 가상 이벤트 리스너 + 프롬프트 템플릿(정적 혹은 경량 매핑)



### 2.3) 장교용 전술 AI 프롬프트 창 (C2 Q&A Chatbot)

지휘소의 장교가 대시보드 화면 옆에서 챗봇 인터페이스를 통해 자연어로 전장 상황 및 교본을 질의할 수 있는 창입니다.

* **기능 요건:**
* **대화형 인터페이스(Chat UI):** 전형적인 생성형 AI 챗봇 창 구현.
* **상황판 연동형 질의응답 (Situational RAG):** * 질의 예시: *"지금 가장 위험한 에지 COP는 어디고 매뉴얼상 대응책은 뭐야?"*
* 답변 예시: *"현재 탄약이 15% 남은 [GOP-21]이 가장 위험합니다. 군수 매뉴얼에 따르면 즉시 우회로를 통해 긴급 보급을 수행해야 합니다."*


* **프라이빗 LLM 인프라 모킹:** 파스업 내부망 환경을 가정하므로 외부 API(OpenAI 등) 대신, 로컬에 띄운 Ollama(Llama-3-8B-Instruct 경량화 버전)를 활용하거나 타임아웃 방지를 위해 사전 정의된 RAG 응답 세트를 Python 백엔드에 결합하여 가속화.


* **추천 기술 스택:**
* **UI:** Streamlit `st.chat_message` 컴포넌트
* **AI Orchestration:** LangChain (초간단 Prompt/Retriever 연결)

## ⏱️ 10시간 개발을 위한 현실적인 Task 분배 (3인 팀 기준)

* **A 개발자 (에지 및 통신 백엔드):** FastAPI + Websocket 기반의 에지 COP 시뮬레이터 개발 및 데이터 송수신 큐 구현 (3시간) ➔ 3단계 명령 동기화 로직 연동 (4시간)
* **B 개발자 (중앙 COP UI 및 지도):** Streamlit을 활용하여 대시보드 레이아웃, Folium 지도 연동, 줌 레벨에 따른 에지 마커 변경 기능 구현 (6시간)
* **C 개발자 (AI 및 컨텍스트 엔진):** 교본 텍스트 데이터 준비, 룰 트리거 연동, Ollama/LangChain을 이용한 챗봇 창 및 추천 액션 프롬프트 엔지니어링 (5시간)
* **전원 합류 (8~10시간):** 데이터 정합성 테스트, 시나리오(탄약 감소 ➔ AI 경고 ➔ 장교 질문 ➔ 명령 하달 ➔ 에지 보급 완료) 리허설 무한 반복.
