# UI 개선 — 팔란티어(Gotham) 스타일 비주얼 레퍼런스 적용

배경: 03.extend_options.md(삭제됨)의 권장 작업 순서("3번 → **UI 개선** → 2번 → 4번") 중 UI 개선 단계의 상세 설계. 04.commands.md와 마찬가지로 별도 문서로 분리한 이유는 대상 파일이 dashboard-web 3개 파일(index.html/app.js/style.css)에 걸쳐 있고, 적용 여부를 항목 단위로 판단해야 하기 때문.

## 참고 원본

- 트윗: https://x.com/babybluecream/status/1995428092719059379 (팔란티어 Foundry/Gotham 아키텍처 스레드, 첨부 이미지가 실제 Gotham COP 화면 스크린샷)
- 스크린샷 요약: 다크 테마 + 위성/지형 지도 배경, 좌측 상황 패널(탭 + 아코디언 상세 섹션 + 심각도 배지 "HIGH"), 우측 "Dossier" 패널(본문 텍스트 안 개체 링크 하이라이트 + 임베드된 관계 그래프(부대→인물→장비 노드-링크) + 선택 개체 속성 카드), 지도 위 객체는 전부 클릭 가능한 아이콘/바운딩박스, 하단 캡션바.
- 핵심은 "지도가 예쁘다"가 아니라 **모든 요소가 클릭 가능한 객체(Object)이고 관계 그래프로 연결되어 있다**는 정보 구조. 단, 이 관계 그래프 자체를 재현하는 것은 아래 트레이드오프 검토에서 범위 제외함.

## 트레이드오프 검토 결과

| 옵션 | 내용 | 장점 | 리스크 |
|---|---|---|---|
| A. 전면 리스킨 | 다크 테마 + 관계 그래프 위젯(부대-장비-명령 링크) + Dossier 스타일 상세 패널 신규 구현 | 시각적 임팩트 최대, 창의성 배점에 강하게 어필 | vanilla JS로 그래프 레이아웃/노드-엣지 렌더링을 처음부터 구현해야 함, 마감(7/5 10:00)까지 남은 시간 대비 디버깅 리스크 큼, CLAUDE.md "완벽한 구현보다 눈에 보이는 것 우선" 원칙과도 배치 |
| **B. 선택적 비주얼 언어 이식 (채택)** | 다크 테마/배지/아코디언 등 기존 DOM·WS 프로토콜을 바꾸지 않는 저비용 항목만 적용 | 순수 CSS + 소규모 JS로 완결, 신규 라이브러리·백엔드 변경 없음, 1시간 내 완료 가능 | 관계 그래프 같은 "우와" 포인트는 없어 창의성 점수 상한이 A보다 낮음 |
| C. UI 미변경 | 남은 시간을 2번(챗봇 하이브리드)·리허설에 집중 | 데모 중 UI 버그 리스크 0 | 화면이 상대적으로 심심해 보이는 인상 리스크 |

**결론: B 채택.** 근거는 03.extend_options.md(삭제됨)와 동일한 원칙 — 마감 임박 시점에 신규 위젯(그래프 엔진)을 붙이는 것은 남은 시간 대비 실익이 적고, 데모 신뢰성만 떨어뜨릴 위험이 큼.

## 현재 상태 대비 격차

이미 확보된 것 (`dashboard-web/style.css`):
- 다크 배경(`body` `#0e1117`), 상태 배지(`.badge-connecting/live/down`), 알림 색상 구분(`.alert-item.critical/warning`), 선택 에지 펄스 링(`.selection-ring`)

없는 것:
- 지도 타일이 기본 OSM 라이트 타일 (`dashboard-web/app.js:26` `L.tileLayer("https://{s}.tile.openstreetmap.org/...")`) — 화면 전체 톤이 다크 UI와 어긋남
- 사이드바 상세 정보가 아코디언 없이 평면 나열 (`dashboard-web/index.html:29-45`의 `.detail-row` 연속 배치)
- 팔란티어 특유의 "classification/military" 느낌을 주는 모노스페이스 폰트, 각진 배지 스타일 부재

## 작업 범위 (dashboard-web만, 백엔드/스키마/WS 프로토콜 변경 없음)

1. **지도 타일 교체** (`app.js:26`) — API 키 불필요한 다크 계열 타일(예: CartoDB `dark_all`)로 교체. 라벨 가독성 확인 후 필요시 라벨 별도 레이어 유지.
2. **사이드바 아코디언화** (`index.html:29-45`) — 기존 `.detail-row` 6개를 3개 그룹으로 재구성: "기본 정보"(ID/좌표/상태), "군수 상태"(탄약 %/진행바), "작전 태세"(무선 상태/다음 갱신/경계 태세). `<details>/<summary>` 네이티브 태그로 구현해 JS 추가 없이 펼침/접힘 확보.
3. **classification 배지 스타일** (`style.css`) — 경계태세 heightened, 탄약 alert 상태에 팔란티어의 "HIGH" 배지 톤(각진 모서리, 굵은 대문자 라벨)을 참고한 배지 클래스 추가. 기존 `.recon-status`, `.alert-banner` 색상 체계는 유지하고 형태만 통일.
4. **모노스페이스 폰트 부분 적용** — 헤더 타이틀, 배지, 라벨 텍스트에 한정해 `"Consolas", "SF Mono", monospace` 적용. 본문(알림 메시지, 챗봇 답변)은 가독성을 위해 기존 산세리프 유지.
5. *(시간 남으면, optional)* 알림 패널(`renderAlert`, `app.js:302`) 항목 클릭 시 해당 `edge_id`의 사이드바 상세로 포커스 이동 — 팔란티어의 "본문 개체 링크 → 우측 상세 패널" 상호작용을 축소 재현. 신규 상태 없이 기존 `selectEdge()` 재사용.

## 하지 않는 것 (Out of scope)

## 완료 기준

대시보드 첫 화면에서 다크 위성 타일 위에 에지 마커가 표시되고, 에지 클릭 시 사이드바가 3개 아코디언 그룹으로 정리되어 보이며, `alert_level=heightened` 또는 `ammo_pct<=20` 상태의 에지가 각진 classification 배지로 한눈에 식별되는지 확인한다. 기존 명령 하달(RESUPPLY 등 6종) 동작과 알림 흐름은 회귀 없이 그대로 동작해야 한다.

## 세션 중 추가 요청 (원 범위를 넘어선 확장 작업)

위 "작업 범위"(B안)를 완료한 뒤, 같은 세션에서 데모 리허설 과정 중 추가로 요청받아 구현한 항목들. 아래는 원래 계획에는 없었던 확장 기록이며, 백엔드/스키마 변경이 포함된 항목도 있어 위 "하지 않는 것" 절과 별개로 진행됨.

1. **클러스터링 제거** — 에지 25개가 넓게 분산돼 있어 클러스터 클릭 시 선택 대신 줌인이 먼저 발생하는 문제가 있어, `leaflet.markercluster` 의존성을 완전히 제거하고 개별 마커를 상시 노출하는 방식으로 전환. CLAUDE.md 컴포넌트 설명도 갱신.
2. **선택 표시 색상 수정** — 다크 타일 배경 위에서 검은색 선택 링이 보이지 않던 문제 → 노란색(`#ffd700`)으로 변경, 갱신 시 `bringToFront()` 추가.
3. **명령 버튼 리스킨** — 기존 둥근 파란 버튼을 classification 배지와 통일된 각진·보더·모노스페이스 스타일로 교체.
4. **시뮬레이터 재시작 기능** — 헤더 우측에 재시작 버튼 추가, 신규 엔드포인트 `POST /api/simulator/restart`(central-backend), `Command`에 `RESET`/`RESET_LOW` 추가. 재시작 시 에지 4개는 의도적으로 경보 수치(critical 2 + warning 2)로 되돌려 알림 데모를 즉시 재현 가능하게 함.
5. **탄약 소모 속도 1.5배 감속** — `edge-simulator/config.py`의 `AMMO_DRAIN_RANGE` `(1,5)` → `(1,3)`.
6. **우회로 정찰(DETOUR_RECON) 명령 추가** — 정찰 결과가 "우회로 권장"(critical)일 때만 노출되는 별도 명령. 데모 단순화를 위해 항상 안전 확인으로 종료하도록 구현.
7. **대시보드 통계 바** — 헤더 하단에 전체/경보/무선침묵/경계태세 4개 카드를 전체 폭으로 배치. 클릭 시 조건에 맞는 에지만 지도에서 강조(나머지는 흐리게 처리)하고 지도 뷰를 자동으로 맞춤(`fitBounds`).
8. **레이아웃 전면 개편** — 지도가 뷰포트의 남는 공간을 전부 채우도록 페이지를 세로 flex 구조로 재구성. "에지 상세 정보"는 지도 위 우측에 항상 표시되는 반투명 오버레이로, "전술 AI 프롬프트"는 지도 위 좌측 토글형 반투명 오버레이로 전환. "최근 알림"은 원래 계획대로 하단 섹션에 유지.
9. **정찰/보급 이동 애니메이션** — 정찰 드론 투입 시 청록색 마커가 보급기지↔에지를 왕복하고, 우회로 정찰 시 주황색 마커가 직선이 아닌 우회 경유점을 지나는 경로로 이동(점선 경로 함께 표시). 우회로 정찰 성공 시 확보한 경유점을 저장해뒀다가, 다음 보급 실행 때 같은 우회 경로로 이동하도록 연결(1회 소비 후 초기화).
10. **버그 수정** — (a) 에지 전환 시 이전 명령 결과 메시지가 그대로 남아있던 문제, (b) 정찰 "안전 확인" 알림이 `AlertEvent.level`이 `warning`/`critical`만 있어 실제 경보와 같은 주황색으로 표시되던 문제(메시지 패턴으로 구분해 프론트에서 초록 `safe` 표시로 보정).

이 확장 작업은 dashboard-web 3개 파일 외에 `central-backend`(main.py/store.py), `edge-simulator`(agent.py/config.py), `shared/schemas.py`, `CLAUDE.md`(엔드포인트 계약 표)까지 걸쳐 있다.
