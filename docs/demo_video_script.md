# 시연 영상 스크립트 (README §6 기준, 3분)

> 촬영 방식: 화면 녹화(OBS 등) + 내레이션(자막 또는 음성). 아래 대사는 그대로 읽거나 자막으로 얹으면 됨.
> 사전 준비: central-backend, edge-simulator 기동 → 브라우저로 대시보드 접속 → 필요 시 `python scripts/demo_scenario.py`로 자동 진행하며 녹화.

## 0:00–0:20 — 오프닝 (문제 정의 + 화면 확인)

**화면**: 대시보드 전체 화면, 지도 위 에지 25개 마커가 5~10초 주기로 갱신되는 모습.

**내레이션**:
> "전방에 흩어진 수십 개의 전초기지, 에지 COP입니다. 폐쇄망·에어갭 환경에서는 이 상태가 상급 지휘부에 실시간으로 모이지 않는 경우가 많습니다. 저희는 이 지휘통제 순환 전체를, 인터넷 연결 없이 로컬 네트워크(WebSocket)만으로 구현했습니다."

## 0:20–0:50 — 위기 발생 (자동 경보 + 교범 매칭)

**화면**: 특정 에지(GOP-21) 탄약을 15%로 강제 설정 → 마커 색 변화, 경보 알림 팝업, 교범 조항 기반 권고문 노출.

**내레이션**:
> "탄약이 20% 아래로 떨어지면 자동으로 경보가 뜨고, 10% 이하는 긴급으로 격상됩니다. 이 판단은 임의 생성이 아니라 실제 교범 조항과 매칭된 룰 엔진이 내립니다."

## 0:50–1:20 — 지휘관 의사결정 지원 (챗봇)

**화면**: 사이드바 챗봇 창에 "가장 위험한 에지는?" 입력 → AI 응답(상황 요약 + 교범 근거) 표시.

**내레이션**:
> "지휘관은 챗봇에 바로 물어볼 수 있습니다. 답변은 실시간 에지 데이터와 교범 조항에 근거해서만 나오도록 설계했습니다."

## 1:20–2:30 — 다중 명령 하달 (핵심 기능 시연)

**화면**: 사이드바에서 순서대로 명령 하달, 매번 지도/마커/상태가 실시간으로 바뀌는 것을 확인.

1. `RECON_DRONE`(정찰 드론) 하달 → 약 6초 후 정찰 결과 알림 브로드캐스트
2. `RESUPPLY`(보급) 하달 → 보급 COP 이동 애니메이션 → 10초 후 보급완료 상태 전환
3. `SILENT_MODE` → `ACTIVE_MODE`(무선 침묵 전환/해제)
4. `ALERT_LEVEL_UP` → `ALERT_LEVEL_DOWN`(경계태세 격상/격하)

**내레이션**:
> "정찰 드론을 띄우고, 보급을 하달하고, 무선 침묵과 경계태세까지 — 버튼 하나로 명령이 에지에 내려가고, 에지 상태가 즉시 지도에 반영됩니다."

(선택) **화면**: "시뮬레이터 재시작" 버튼 클릭 → 전 에지 초기화, 일부 에지는 즉시 탄약 경보 재현.

## 2:30–3:00 — 마무리 & 로드맵

**화면**: README 아키텍처 다이어그램(§3-1 확장 아키텍처) 또는 슬라이드 컷.

**내레이션**:
> "지금 보신 것은 파스업이 디딤돌 R&D 과제로 개발 중인 Edge DIP, 그리고 업스테이지와 협업 중인 국방 특화 LLM Agent의 축소 프로토타입입니다. 실전 배치 시 이 인메모리 코어는 파스업 DIP·Edge DIP의 MQTT/Kafka·StarRocks/Iceberg·vLLM+Qdrant/Milvus 파이프라인으로 그대로 확장됩니다."
